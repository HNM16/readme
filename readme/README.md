## Что такое Git?

**Git** — это распределённая система контроля версий, которая позволяет отслеживать изменения в коде и работать над проектами как в одиночку, так и в команде. Git хранит историю изменений, поддерживает ветвление и слияние, а также предоставляет возможность откатиться к предыдущим версиям проекта.

### Основные особенности Git:
1. **Контроль версий**: Позволяет сохранять все изменения в проекте и восстанавливать их при необходимости.
2. **Ветвление и слияние**: Разрабатывайте новые функции независимо друг от друга и объединяйте их в основную ветку.
3. **Распределённая архитектура**: Каждый разработчик имеет полную копию репозитория, что исключает зависимость от центрального сервера.
4. **Высокая производительность**: Git эффективно работает с проектами любого размера.

### Преимущества Git:
- Совместная работа над проектом без конфликта изменений.
- Возможность экспериментировать в изолированных ветках.
- Обратимость изменений для устранения ошибок.
- Интеграция с платформами, такими как GitHub, для упрощённого взаимодействия.

### Базовый рабочий процесс:
1. Измените файлы.
2. Добавьте изменения в индекс с помощью `git add`.
3. Зафиксируйте изменения с помощью `git commit`.
4. Отправьте изменения в удалённый репозиторий с помощью `git push`.

## Что такое GitHub?
![Логотип GitHub](/GitHub-logo.png)  


**GitHub** — это веб-платформа для хостинга Git-репозиториев. Она дополняет функциональность Git инструментами для совместной работы, такими как pull request, управление задачами (issues) и средства автоматизации. GitHub позволяет разработчикам со всего мира делиться своими проектами и вносить вклад в открытые репозитории.

### Основные особенности GitHub:
1. **Удалённые репозитории**: Хранение вашего кода в облаке.
2. **Pull Requests**: Возможность обсуждать и проверять изменения перед их объединением.
3. **Управление задачами**: Отслеживание ошибок, предложений и улучшений.
4. **Автоматизация**: Интеграция CI/CD с помощью GitHub Actions.
5. **Сообщество**: Возможность делиться проектами, вносить изменения в чужие репозитории и участвовать в разработке open-source.

### Преимущества GitHub:
- Надёжное хранение резервных копий вашего кода.
- Совместная работа с глобальными командами.
- Публикация открытых репозиториев в качестве портфолио.
- Участие в open-source проектах для обучения и обмена опытом.
